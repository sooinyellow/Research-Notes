## 컴포넌트 기반 개발 (CBD, Component Based Development) 방법론

1. 컴포넌트: 재사용 가능한 독립적인 구성단위
2. 인터페이스: 컴포넌트가 수행할 하나 이상의 오퍼레이션의 집합.
- 인터페이스를 통해서, 컴포넌트 개발자와 사용자가 상호작용이 가능해짐.(따라서, 안정 + 변화가 적어야함)
- 인터페이스가 유지되면 컴포넌트 구현은 바뀌어질 수 있음.
- 하지만, 컴포넌트 내부가 은닉될 수 있음.
3. 컴포넌트 기반 개발: 전체 소프트웨어를 만드는 것이 아니라 개별로 만드는 것
- 도메인 분석 -> 도메인 설계 -> 컴포넌트 추출 -> 컴포넌트 설계 -> 컴포넌트 구현 -> 컴포넌트 인증 -> 분류/형상관리
- 기술 아키텍처를 알아야함.
- 컴포넌트는 재사용 목적 (따라서, 도메인에 대한 분석이 매우 중요)
4. 컴포넌트 기반 소프트웨어 개발: 개발된 컴포넌트를 조립해서 특정 소프트웨어를 개발하는 것
- (어플리케이션 요구사항) -> (CBD 설계) -> 컴포넌트 특화 -> 컴포넌트 조립 -> 새로운 어플리케이션
